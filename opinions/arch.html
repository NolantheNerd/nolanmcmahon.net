Arch Linux Installation

Arch Linux is a distribution of GNU/Linux which is very minimal. It takes effort to both install it and configure it. This can take time, but allows you to more easily configure your system to be exactly how you want it to be. This set of instructions here will take you quickly through the entire process. Note that this is more of a quick reference than a guided tour. I would recommend other sources if you truly want to understand why each command is necessary/desirable.

The Absolute Basics

This section gets you to the bare minimum: Arch Linux in its default form without any bells or whistles. By the end of this section, you should have a black screen with a command prompt.

  1. First, create a USB drive which contains a bootable copy of Arch Linux. Best to go to the Arch Linux Website to get a copy of the ISO file.
  2. Put the ISO on your USB. If you are running Windows or MacOS, I would encourage you to use Fedora's Media Writer to configure your USB device with the ISO. On any version of GNU/Linux, dd is your best bet. dd if=(ISO LOCATION) of=/dev/(USB DRIVE) status="progress"
  3. Boot the machine from the USB, usually by pressing F2 or F11 or F12 as the machine boots and choosing the Arch Linux USB.
  4. Check for UEFI. If the output of ls /sys/firmware/efi/efivars is ls: cannot access '/sys/firmware/efi/efivars': No such file or directory then you can proceed to install Arch in this way. Otherwise, you need to install Arch Linux for UEFI systems. Search for that and you'll find a guide that can help you.
  5. Make sure you have an internet connection. ping nolanmcmahon.net
    64 bytes from nolanmcmahon.net ...     With an Ethernet cord, you should be connected automatically, otherwise you will need to connect with WIFI.
  6. If you want to connect with WIFI, first enter the iwd command line interface with iwctl. List your WIFI hardware with device list. Make sure that the device that you want to connect with is not disabled/turned off (with a hardware switch on the keyboard/chassis). Note the name of your device. Connect to your WIFI network with: station (name of your device) connect (name of your wifi network)Enter your WIFI password and then type exit to return to the regular command prompt. Double check that you are connected to the internet with the above ping command.
  7. Run: timedatectl set-ntp true
  8. Use: lsblk to find the name of the drive on which you want to install Arch.
  9. Start partitioning your drive with fdisk /dev/(name)
  10. List partitions with p.
  11. Delete all partitions with d.
  12. Create a boot partition with n (Make Primary if Prompted, Partition Number Default, First Sector Default, Last Sector +200M, Clear Signature if Prompted).
  13. Create swap partition with n (Make Primary if Prompted, Partition Number Default, First Sector Default, Last Sector +(150% RAM), Clear Signature if Prompted).
  14. Create a root partition with n (Make Primary if Prompted, Partition Number Default, First Sector Default, Last Sector +25G (absolutely no less than 15), Clear Signature if Prompted).
  15. Create a home partition with n (Make Primary if Prompted, Partition Number Default, First Sector Default, Last Sector Default, Clear Signature if Prompted).
  16. Write changes with w.
  17. Make file systems on boot, root, and home with: mkfs.ext4 /dev/(name)1
    mkfs.ext4 /dev/(name)3
    mkfs.ext4 /dev/(name)4
  18. Make swap partition with: mkswap /dev/(name)2
  19. Turn on swap with: swapon /dev/(name)2
  20. Mount the root partition with: mount /dev/(name)3 /mnt
  21. Make folders upon which to mount the remaining partitions with: mkdir /mnt/home /mnt/boot
  22. Mount the boot and home partitions with: mount /dev/(name)1 /mnt/boot and mount /dev/(name)4 /mnt/home
  23. Install Arch Linux with: pacstrap /mnt base base-devel linux linux-firmware vim
  24. Make an fstab file with: genfstab -U /mnt >> /mnt/etc/fstab Correct the file if necessary.
  25. Move to the drive with: arch-chroot /mnt
  26. Install Network Manager with: pacman -S networkmanager
  27. Enable the Network Manager on startup with SystemD with: systemctl enable NetworkManager
  28. Install the boot loader with: pacman -S grub
  29. Install grub with: grub-install --target=i386-pc /dev/(name)(often times --force is required to get grub to install properly)
  30. Configure grub with: grub-mkconfig -o /boot/grub/grub.cfg
  31. Make a root password with passwd.
  32. Configure languages using: vim /etc/locale.gen and uncommenting the lines relevant to you (ie: en_CA/en_US; there should be two lines per language)
  33. Run: locale-gen
  34. Set the lang variable with: vim /etc/locale.conf and add the line LANG=en-CA.UTF-8 or LANG=en-US.UTF-8
  35. Set your timezone. Look for time zones with: ls /usr/share/zoneinfo/ and working your way down. Link with: ln -sf /usr/share/zoneinfo/(YOUR TIME ZONE) /etc/localtime
  36. Give your machine a host name with vim /etc/hostnameAdd one word at the top of the file with the name you wish to give your machine.
  37. If you want to use WIFI, install iwd with: pacman -S iwd
  38. Run exit. Then unmount the drives with: umount -R /mnt You can now safely reboot with reboot.
  39. Remove the USB drive from the system after it shuts off; it should boot from your internal hard drive.

Making Arch Usable

Now, if you did everything correctly, you should have a command prompt on a black screen. Next lets start to add features to the system to allow you to actually use it.

  1. Login with the username root and the password you set before.
  2. If you are using WIFI, we need to set it up to connect. First enable the services that WIFI needs:systemctl enable --now systemd-networkd systemd-resolved iwdThen the config for the wireless connection needs to be edited. To the new file /etc/systemd/network/20-wireless.network add the following lines:[Match]
    Name=wlan0

    [Network]
    DHCP=yes    Now run: systemctl restart systemd-networkdLastly, enter the iwd command prompt with iwctl and connect to the WIFI network using station (name of your device) connect (name of your wifi network) and type exit to return to the regular prompt. Check to make sure that you are connected to the internet with the ping command.
  3. Add a user account with: useradd -m -g wheel (username) and give them a password with: passwd (username)
  4. Run: sudo vim /etc/sudoers and uncomment the line: %wheel ALL=(ALL) NOPASSWD: ALL which is near the bottom of the file.
  5. Install basic fonts with: pacman -S noto-fonts
  6. Many programs (like st) will need to know what fonts are available. They need fontconfig. pacman -S fontconfig
  7. Install git with: pacman -S git
  8. Make a directory for all of the configs you will need for your system. In /home/(username) run: mkdir .config
  9. Make a directory for all of the software that you will first git clone, then make install. I like ~/software/. To do that: mkdir softwarein ~/.
  10. It's strictly optional, but I like to make pacman a bit more fun, so I add ILoveCandy and uncomment Color in /etc/pacman.conf.
  11. Being able to read the manual is essential. To allow this, install man-db. pacman -S man-db
  12. Create a directory for scripts you'll use. I like ~/scripts/. mkdir scripts
  13. I want the scripts in this folder to be a part of my system path. To do this add: export $PATH:$HOME/scripts to .bashrc.
  14. I like to create the "traditional" directories for all of my personal files, in particular mkdir Documents Downloads Pictures Music Videos
  15. To solve an issue where pacman can't grab PGP keys when downloading software, install:pacman -S archlinux-keyring

AUR Helper - yay

It's likely that not all of the software that you'll want will be available in the official Arch Linux repositories. This is where the Arch User Repository (AUR) comes in! To make it easier to use it, I use Yet Another Yogurt (YAY) to download, install, and manage software from the AUR.

  1. I use yay as my AUR helper. In ~/software/ clone the yay repository: git clone https://aur.archlinux.org/yay.git
  2. Move into yay repository and execute: makepkg -sri

X Org Server

The next step is setting up a graphical interface. There are two main options: X and Wayland. I prefer X.

  1. To get a graphical environment you need X. Run: pacman -S xorg-server
  2. You need an initialization file for X. I like xinit so create ~/.xinitrc and install: pacman -S xorg-xinit
  3. You need to make sure that you have the correct drivers for your graphics card. Run this command to find the name of the graphics card you have: lspci -v | grep -A1 -e VGA -e 3D
  4. Then install the appropriate driver from the list on the Arch Wiki. Warning! For multiple monitor set-ups with nVidia cards, the open source driver hasn't worked well for me. It would not allow me to permute my monitors without revealing some nasty bugs. In which case, I recommend the proprietary video drivers from nVidia: pacman -S nvidia nvidia-utils nvidia-settings
  5. To get information about your X server, you'll want xdpyinfo. pacman -S xorg-xdpyinfo
  6. For multiple monitors, you'll want xrandr. pacman -S xorg-xrandr
  7. To configure multiple monitors, first find their names with xrandr -q. They should be something like HDMI-1, DP-1 DVI-D-1, LVDS, VGA-1, etc.
  8. You can arrange your monitors in many different ways, but I prefer to keep it simple. Modify this line to match what you configuration is: xrandr --output DP-1 --primary --auto --output DVI-D-1 --auto --left-of DP-1 --output HDMI-1 --auto --right-of DP-1Alternatively, this can be changed in nvidia-settings if you are using the nVidia video driver.
  9. For transparency support, install xcompmgr using: pacman -S xcompmgr
  10. xcompmgr needs to be started for transparency to work. Add: xcompmgr -c & to .xinitrc.
  11. For wallpaper configuration, I use xwallpaper. Install using: pacman -S xwallpaper
  12. Once I have a wallpaper that I like, I want it to be persistent. To do this, add: xwallpaper --zoom Pictures/wall.jpg to .xinitrc. This will make the file ~/Pictures/wall.jpg your wallpaper each time you login.
  13. I don't like my screen blanking after a period of inactivity. To prevent that add the following three lines to .xinitrc:xset s off # Don't Activate Screensaver
    xset -dpms # Disable DPMS (Energy Star) Features
    xset s noblank # Don't Blank the Video Device
  14. I want X to start automatically after logging in without having to provide startx. To do this, add: if [[ "$(tty)" = "/dev/tty1" ]]; then
    startx
    fi    to your .bashrc.
  15. I want to be able to automatically change my cursor on the desktop to the familiar pointer instead of the default X, to do this I need: xsetroot: pacman -S xorg-xsetrootThe change itself is made in the Window Manager section below.
  16. I like to have a consistent appearance across my system (of mostly terminal-based programs). To do this, I use Xresources. pacman -S xorg-xrdb
  17. Add your configurations to ~/.Xresources. See my dotfiles for how I configure Xresources.
  18. The font I'm currently using in my Xresources config is Iosevka: yay -S ttf-iosevka
  19. Load the ~/.Xresources file in .xinitrc by adding the line: [[ -f ~/.Xresources ]] && xrdb -merge -I$HOME ~/.Xresourcesto your .xinitrc.

Keyboard Configuration Script

The keyboard has a glaring problem: Caps Lock is a USELESS key in a GREAT location. I use Esc and SUPER all the time, so it makes sense to remap Caps Lock to both of those keys. This section will configure Caps Lock to act like Esc if it is pressed in isolation, but act like SUPER if pressed in combination with any other key! The script also makes all key presses register in quicker succession if a key is held down.

  1. Copy from my dotfiles the script called remap into scripts.
  2. The script depends on xdotool and xcape. Install them: pacman -S xcape xdotool
  3. Add ~/scripts/remap to .xinitrc.

Window Manager - BSPWM/SXHKD

Interacting with programs in a graphical environment requires some form of window manager. I vastly prefer a tiling window manager to a floating one, so I'm using BSPWM. This section will install and configure BSPWM and setup shortcuts for interacting with the windows that it creates.

  1. Install the window manager BSPWM with: pacman -S bspwm and a hotkey tool (highly recommended by BSPWM): pacman -S sxhkd
  2. In ~/.config/: mkdir bspwm
  3. Copy the default BSPWM config using: cp /usr/share/doc/bspwm/examples/bspwmrc ~/.config/bspwm/bspwmrc
  4. If you have multiple monitors, set the workspaces you want on each one with something like: bspc monitor (monitor name from xrandr -q) -d VII VIII IX
  5. In ~/.config/: mkdir sxhkd
  6. Copy the default SXHKD config using: cp /usr/share/doc/bspwm/examples/sxhkdrc ~/.config/sxhkd/sxhkdrc
  7. Configure your bspwmrc and sxhkdrc files as desired. My configs are available in my dotfiles.
  8. To bspwmrc I like to add:xsetroot -cursor_name left_ptr
    bspc config focused_border_color \#fcc200
    bspc config normal_border_color \#000000
    bspc config focus_follows_pointer true
    bspc config pointer_follows_focus true
  9. Add exec bspwm to the very bottom of your .xinitrc

Status Bar - polybar

BSPWM only manages windows; it does not come with its own status bar. You have to add that yourself. Polybar is the program that I've gone with for creating status bars.

  1. I like to use polybar as my status bar of choice. polybar is an AUR package so it is installed with: yay -S polybar
  2. Look to my dotfiles for how I configure my polybar. Set the config in ~/.config/polybar/config.
  3. Fonts are a big issue with Polybar. You'll need several: xorg-fonts-misc pacman -S xorg-fonts-miscUnifont pacman -S bdf-unifontSiji yay -S siji-gitSymbola yay -S ttf-symbola-free
  4. If you configure it to use a unicode character that is not available in the fonts that are loaded by polybar, you can search for fonts that have the unicode character that you need at fileformat.info
  5. If for some reason fonts break, check to see if running fc-list as root lists all necessary fonts. If it does, copy the directory of locked fonts into the same fonts folder with a different name. Problem "solved".

Bluetooth - bluez

Bluetooth is pretty handy if you want to connect a mouse, keyboard, or other input device. Here's how I set it up.

  1. To enable bluetooth, you first must have hardware which permits it, then install bluez and bluez-utils. pacman -S bluez bluez-utils
  2. To connect a bluetooth device (NOT AN AUDIO DEVICE - see Arch wiki for additional steps to use Audio bluetooth) run: bluetoothctl
    power on
    scan on
    agent on
    pair MAC-ADDRESS (choose a MAC address from tab select at random if you don't know which one is your device's address)
    connect MAC-ADDRESS
  3. If you want the bluetooth connection to be made automatically at startup, you have to automatically power on the bluetooth device by changing /etc/bluetooth/main.conf to have AutoEnable=true under the [Policy] section.

Application Launcher - DMenu

What good are programs if you can't launch them? I've chosen dmenu to be my program launcher.

  1. To launch applications, I prefer dmenu because it is minimal. Install with: pacman -S dmenu

Terminal Emulator - st

No GNU/Linux system is complete without a terminal! I use the simple/suckless terminal from suckless.org with several patches.

  1. st requires libxft. Install using: pacman -S libxft
  2. Clone the st repository from suckless.org. In your ~/software/ directory run: git clone https://git.suckless.org/st
  3. Next download the patches that you want to apply to st. I like to keep them in the directory for the st itself so inside of ~/software/st: mkdir patches
  4. I like having support for transparency in my terminal so I use the Alpha patch. Inside ~/software/st/patches run: curl -s http://st.suckless.org/patches/alpha/st-alpha-0.8.2.diff > alpha.diffto get Alpha version 0.8.2.
  5. I like being able to scroll back in the terminal, so I use scrollback. Inside ~/software/st/patches run: curl -s http://st.suckless.org/patches/scrollback/st-scrollback-20210507-4536f46.diff > scrollback.diffto get a recent version of scrollback.
  6. I also like being able to scrollback with the mouse, not just the keyboard. Inside of ~/software/st/patches run: curl -s http://st.suckless.org/patches/scrollback/st-scrollback-mouse-20191024-a2c479c.diff > scrollback_mouse.diff
  7. I want st to respond to the configs that I set in .Xresources. Inside of ~/software/st/patches run: curl -s http://st.suckless.org/patches/xresources/st-xresources-20200604-9ba7ecf.diff > xresources.diff
  8. Apply all your desired patches by running: patch -i patches/(patch_name)in the ~/software/st directory. (One patch call per patch)
  9. Now you want to configure st (including the patches you applied). In ~/software/st run: make config.hEdit config.h to your heart's content.
  10. In particular, I like to be able to scroll in st with Alt+[jkud] like in Vim, so I add the following lines into the static Shortcut shortcuts[] variable:{ MODKEY, XK_j, kscrolldown, {.i = 1} },
    { MODKEY, XK_k, kscrollup, {.i = 1} },
    { MODKEY, XK_u, kscrollup, {.i = -1} },
    { MODKEY, XK_d, kscrolldown, {.i = -1} },
  11. When you are ready to install it: sudo make install
  12. I want st to be my default terminal. To do this add: export TERMINAL="st"to .bashrc.

Web Browser - qutebrowser

A web browser is absolutely essential. I use qutebrowser. It is a very simple, configurable browser with Vim shortcuts built in.

  1. My web browser of choice is qutebrowser. To install it: pacman -S qutebrowser
  2. Type :open qute://settings/ to go to the settings page for qutebrowser:Set tabs.last_close to close
    Set completion.cmd_history_max_items to 0
    Set completion.web_history_max_items to 0
  3. I want qutebrowser to be my default web browser. To do this add: export BROWSER="qutebrowser" to .bashrc.

Text Editor - Neovim

Vim, but with fewer lines of source code! It's my IDE of choice.

  1. My text editor and development environment of choice is Neovim. Install using: pacman -S neovim
  2. Create the directory ~/.config/nvim/.
  3. Create an init file in ~/.config/nvim/ with: touch init.vimin that directory.
  4. Download vim plug, to be the plugin manager for Neovim with: curl -fLo ~/.config/nvim/autoload/plug.vim --create-dirs https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
  5. Make a new directory for the plugins with: mkdir: ~/.config/nvim/vim-plug
  6. Create a file in the vim-plug directory to pull the plugins that are desired. See my dotfiles for how I structure this file.
  7. Source the plugins file in the init.vim file. For details on how I do this, take a look at my init.vim dotfile.
  8. Install the given plugins by opening up Neovim and running :PlugStatus and then :PlugInstall. See this article for more details.
  9. Lastly, I want Neovim to be my default text editor on the system. To do that add: export EDITOR='nvim' to .bashrc.

Mail Client - Neomutt

All your emails from all your accounts in one place, locally stored on your own computer and accessible from a command line interface. What more could you ask for?

  1. I use mutt-wizard to configure neomutt with isync. This allows me to have all of my mail offline. First install required dependencies: pacman -S isync neomutt msmtp pass
  2. mutt-wizard has a number of optional dependencies. If you want to be able to view HTML in neomutt, install lynx: pacman -S lynx
  3. If you want to index your mail, install notmuch. pacman -S notmuch
  4. If you want to have urls in email output to your browser, you'll need urlview: yay -S urlview
  5. If you want an address book to suggest email address completion, you'll need abook: yay -S abook-configdir
  6. Now, to install mutt-wizard:git clone https://github.com/LukeSmithxyz/mutt-wizard
    cd mutt-wizard
    sudo make install
  7. Make a gpg key pair with: gpg --full-generate-key
  8. Initialize pass with: pass init (your gpg email)
  9. Now add accounts using: mw -a (your email address)
  10. Sync mail with: mw -y (your email address)
  11. Mail can now be viewed by running neomutt.

File Viewer - ranger

Ranger is my terminal-based file viewer of choice. It's simple and it works well.

  1. I like the command line file viewer: ranger. To install: pacman -S ranger
  2. To configure ranger, run: ranger --copy-config=allThis moves all the config files into ~/.config/ranger.
  3. In rc.conf, add: RANGER_LOAD_DEFAULT_RC FALSE
    set show_hidden true

Music Player - ncmpcpp/mpd

Music from the terminal! NCMPCPP also has a visualizer mode which introduces another way to groove to your tunes.

  1. I use Pulse Audio as an audio server. To install: pacman -S pulseaudio
  2. I control Pulse Audio with pamixer. To install: pacman -S pamixer
  3. I like to use the mpd with ncmpcpp as a front end. To install: pacman -S mpd ncmpcpp
  4. I use a per-user config for mpd. Create ~/.config/mpd with: mkdir mpdin the ~/.config/ directory.
  5. Reference my dotfiles to see what I include in ~/.config/mpd/mpd.conf.
  6. I just want mpd to run at startup, so I add mpd ~/.config/mpd/mpd.confto my .xinitrc
  7. To setup configs: cp /usr/share/doc/ncmpcpp/config ~/.config/ncmpcpp/config
    cp /usr/share/doc/ncmpcpp/bindings ~/.config/ncmpcpp/bindings
  8. Check out my dotfiles for how I change these configs.

Image Viewer - sxiv

SXIV is simple. It does everything you need it to and nothing you don't. It gets out of the way as much as possible, which is excellent.

  1. I like to use the Simple X Image Viewer. To install: pacman -S sxiv

Video Player - mpv

Similarly to SXIV, MPV is a video player which is full of all the features you need with none that you don't.

  1. I use mpv to play video files. Install with: pacman -S mpv

PDF Reader - Zathura

Zathura is a PDF (and PS and DJVU and Comic Book) reader which prioritizes letting you view documents. No long load times, or laggy scrolling. And it certainly doesn't nag you to upgrade to a premium, paid subscription (cough cough ADOBE cough cough).

  1. The PDF/Postscript file viewer that I like is Zathura. Install the Zathura base with: pacman -S zathura
  2. For PDF support install either the mupdf backend or the poppler backend. I personally use the mupdf backend. To install: pacman -S zathura-pdf-mupdf
  3. If you want PostScript support, install the PostScript backend. pacman -S zathura-ps
  4. I want Zathura to follow the tiled layout, so I add: bspc rule -a Zathura state=tiled to my bspwmrc.

Calculator - bc

No fancy buttons or graphical elements, just a really simple calculator program.

  1. I like the command line calculator: bc. To install: pacman -S bc

Mathematical Analysis Software - SageMath

When a basic calculator won't cut it (like when you need to numerically solve an integral) Sagemath is a FLOSS go to alternative to Mathematica and Maple.

  1. Sagemath is my go to for mathematical analysis. To install: pacman -S sagemath sagemath-doc

Financial Accounting Software - ledger

To keep track of my personal finances, I use ledger.

  1. I use ledger to manage my personal finances. Install with: pacman -S ledger
  2. Ledger comes with the ability to plot your finances with GNUPlot. Install with: pacman -S gnuplot
  3. To actually use the plotting features, you'll need to add the report script to your scripts folder, see my dotfiles.

Messaging - Slack, Discord

Communication with other people. Gotta hop on the proprietary train because everyone else is already there.

  1. Proprietary software, but I need it for work. Open source desktop clients "exist" although many of them are no longer supported or have many of the features you may need from Slack. I bite the bullet and use the official package: yay -S slack-desktop
  2. Proprietary software, but everyone I know uses it. You may be able to use a third-party open source client, but then you are violating the terms of service with Discord, and risking getting banned. Use at your own risk. pacman -S discord
